# AIWorkspace 重构设计文档（V2）

## 项目目标

将当前 `AISkillManager` 重构为：

```text
AI Workspace
```

从“Skill 管理器”升级为：

- 多 Agent 统一管理
- MCP 统一管理
- Prompt / Workflow / Rule / Context 管理
- 多 IDE / 多 Agent Adapter
- 项目级 AI Workspace
- AI 能力装配系统

目标：

```text
One Workspace
Multiple AI Agents
Multiple MCP Servers
Unified Capability Management
```

------

# 一、核心设计理念

## 1. Source Of Truth

项目中的：

```text
.ai/
```

作为唯一真实配置源。

所有：

- Cursor
- Claude
- Continue
- Antigravity
- Windsurf
- Gemini CLI
- Codex

配置均由 `.ai/` 自动生成。

即：

```text
.ai -> adapters -> target IDE/Agent
```

而不是手工维护：

```text
.cursor/
.cline/
.continue/
```

------

## 2. Workspace First

以“项目工作区”为核心。

而不是：

```text
用户全局 prompt
```

每个项目：

- 独立 MCP
- 独立 workflow
- 独立 context
- 独立 profile
- 独立 rules

------

## 3. Capability-Based Architecture

不再以：

```text
Skill
```

为核心。

而是：

```text
Capability
```

即：

- MCP
- Prompt
- Workflow
- Context
- Rule
- Tool

共同组成 AI Capability。

------

## 4. Adapter Layer

不同 AI 工具只是 Adapter。

例如：

| Tool        | Adapter            |
| ----------- | ------------------ |
| Cursor      | CursorAdapter      |
| Claude      | ClaudeAdapter      |
| Continue    | ContinueAdapter    |
| Antigravity | AntigravityAdapter |

核心数据结构统一。

# 三、模块设计

------

# 1. agents/

## 作用

定义 Agent 行为与能力。

例如：

- Claude Code Agent
- Cursor Coding Agent
- Architecture Agent
- Review Agent
- Documentation Agent

------

## 示例

```text
.ai/agents/cpp-reviewer.json
{
  "name": "cpp-reviewer",
  "description": "C++ code review agent",
  "prompts": [
    "cpp-review"
  ],
  "rules": [
    "cpp-style"
  ],
  "workflows": [
    "review-workflow"
  ],
  "profiles": [
    "cpp"
  ]
}
```

------

# 2. mcp/

## 作用

统一管理 MCP Servers。

支持：

- stdio
- sse
- streamable-http

支持 Adapter 自动转换。

------

## 示例

```text
.ai/mcp/oceanengine.json
{
  "name": "oceanengine",
  "transport": "sse",
  "command": "mcp-remote",
  "args": [
    "${OCEANENGINE_MCP_URL}",
    "--transport",
    "sse-only"
  ],
  "capabilities": [
    "ads-api",
    "marketing"
  ]
}
```

------

# 3. prompts/

## 作用

Prompt Library。

支持：

- system prompt
- task prompt
- reusable prompt
- chain prompt

------

## 示例

```text
.ai/prompts/qt-code-review.md
```

------

# 4. workflows/

## 作用

定义 AI Workflow。

例如：

- Code Review
- Bug Fix
- PR Analysis
- Dump Analyze
- SDK Doc Generate

------

## 示例

```text
.ai/workflows/sdk-doc-generate.json
{
  "steps": [
    "parse-header",
    "extract-api",
    "generate-markdown"
  ]
}
```

------

# 5. rules/

## 作用

定义编码规则与行为规则。

例如：

- C++ Style
- Git Commit Rule
- Markdown Rule
- Architecture Constraints

------

## 示例

```text
.ai/rules/cpp-style.md
```

------

# 6. context/

## 作用

项目上下文。

例如：

- 技术栈
- 架构说明
- API 文档
- 领域知识
- 企业规范

------

## 示例

```text
.ai/context/project.md
```

------

# 7. profiles/

## 作用

能力组合配置。

类似：

```text
VSCode Profile
Docker Compose
```

------

## 示例

```text
.ai/profiles/qt.json
{
  "prompts": [
    "qt-review",
    "cmake"
  ],
  "rules": [
    "cpp-style"
  ],
  "mcp": [
    "github",
    "filesystem"
  ]
}
```

------

> 最后更新：2026-05-19

## 项目初心

统一各种 AI Agent 工具（Cursor / Claude / Gemini / Windsurf / Codex / …）的 skill、rule、mcp 配置，以 `~/.ai/` 为全局唯一源，通过 junction/symlink 同步到各 Agent 配置目录。

```
One Workspace → Multiple AI Agents → Unified Capability Management
```

## 二、版本路线图

## 二、架构定位与核心原则 (Core Architecture & Principles)

**核心定位**：AI Capability Package Manager (无状态配置包管理器 / AI Capability Workspace)。
**非目标 (Non-Goals)**：坚决**不**做 AI Workflow Runtime，**不**接管 Agent 执行状态。

**三大核心原则**：
1. **绝对无状态 (Stateless)**：只负责配置的静态分发，拒绝执行图 (DAG)、状态管理 (Memory/Session)。
2. **渐进增强 (Progressive Enhancement)**：
   - Level 0 (零门槛)：普通单文件 `skill.md`，默认无外部依赖，兼容全 Agent。
   - Level 1 (可选元数据)：支持读取文件顶部的 YAML Front Matter 定义依赖和路由规则。
   - Level 2 (旁路/覆盖机制)：针对第三方托管（如 Git Submodule）的外部 Skill，采用 Profile 集中覆盖 (Override) 机制处理，保证外部文件只读且可顺畅 Pull 更新。
3. **Core IR + Extension (Adapter机制)**：提取各 Agent 的“最低公共分母”作为基础能力 (Core IR)，对 Cursor/Claude 等的专有高级特性，通过 `extensions` 字段直接透传 (Passthrough)，避免能力降级。

## 三、V1.0 开发任务（目标：极致的个人效率引擎）

### P0 — 核心目标（最高优先级）

#### 1. 全面铺开目标适配 (Target Extension)
**目标**：解决“空间复用”问题，实现一次编写，到处运行。
**方案**：
- 在系统核心路由层硬编码注册所有主流 AI 助手的配置路径，实现配置的一键全量打通（通过符号链接或生成覆盖）。
- **必须支持的核心端**：Cursor, Gemini/Antigravity, Claude, Codex, Trae, Qoder, Astrbot, DeepSeek TUI, Windsurf, Continue 等。

#### 2. 极简 Profile 引擎 (Profile Engine)
**目标**：解决“时间复用”问题，应对不同项目类型（前端/后端）需要不同能力集的痛点。
**方案**：
- 在 `~/.ai/profiles/` 下管理纯文本/YAML清单（如 `frontend.yaml`、`rust.yaml`）。
- **组合装配**：支持在 Profile 中自定义组合所需的 skills、rules，或者直接新建专有的规则映射。
- **UI 交互**：新增“Profile 下拉切换”组件，选中后瞬间完成全盘勾选状态的切换，并自动向所有受管 Agent 触发同步。

#### 3. MCP Core IR 与 Adapter 渲染机制
**目标**：解决 MCP 格式碎片化导致的无法直接共享问题。
**方案**：
- 定义统一的 `mcp.yaml` 作为中间层 (Core IR)，仅抽象基础命令。
- 设计 `Adapter` Trait 引擎，按目标 Agent 将 `mcp.yaml` 编译为各自原生的 JSON。保留 `extensions` 字段用于直接透传专属高阶配置（Passthrough）。

### P1 — 体验完善

#### 4. 解析器渐进增强与 Override 接管
- 支持读取 `.md` 顶部的 YAML Front Matter 获取依赖元数据。
- 支持在 Profile 中直接声明 Override 规则强行覆盖第三方能力包的元数据，保障 Git Submodule 等外部引入手段的安全更新与零冲突。

#### 5. 冲突检测与安全备份机制
由于 Profile 切换会带来全量单向覆盖，为了防范隐式覆盖掉用户在 Agent 端手改的内容，必须提供严格的备份策略（加 `.bak`）及状态脱轨（State Drift）提示。

#### 6. 配置增删管理 & 全局搜索
在 UI 界面支持直接创建/删除 Skill、Rule 及 MCP，加入跨分类的全局关键字搜索定位。

## 四、V2.0 储备池（按需探索）

V2 阶段仍然坚决不碰 Runtime 和执行编排，仅关注静态配置的进一步延伸。

### 1. Capability Packaging (导入与导出)
支持将特定 Profile 及其挂载的所有 skill/rule/mcp 打包，通过命令行（如 `aiws export frontend-profile`）导出，用于跨机同步或团队内分享。

### 2. 项目级 `.ai/` 工作区
支持从全局 `~/.ai/` 下放权限到当前项目的代码仓库根目录 `.ai/`，实现针对具体项目的强制性环境隔离。

## 五、优先级总览

```
P0（核心生产力，立刻执行）
├─ 全面铺开目标适配（Cursor/Antigravity 等十余款 Agent 支持）
├─ 极简 Profile 引擎（配置清单管理与 UI 一键切换）
└─ MCP Core IR 与 Adapter 渲染引擎

P1（体验闭环机制）
├─ 渐进增强解析器与第三方 Override
├─ 强力冲突检测与安全备份体系
└─ UI 内建增删能力与全局搜索

V2（储备探索区）
├─ Capability Import/Export 打包系统
└─ 项目级 .ai/ 支持
```
